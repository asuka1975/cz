# Rule: 仕様ドキュメントの規約

## Priority: HIGH

## ファイル命名

仕様ドキュメントは `docs/spec/` 配下に以下の命名規則で配置する:

```
docs/spec/NN-<topic>.md
```

- `NN` は 2 桁の連番 (00, 01, 02, ...)
- `<topic>` はケバブケースの英語トピック名

## ドキュメント構成

各仕様ドキュメントには以下のセクションを含めること:

1. **概要** - この仕様が定義する内容の簡潔な説明
2. **構文** - BNF またはそれに準ずる形式での構文定義 (該当する場合)
3. **意味論** - 動作の詳細な説明
4. **例** - Cz コードによる具体例
5. **制約・制限** - 制約事項や未定義動作

## 言語

仕様ドキュメントは日本語で記述する。コード例やBNF定義内の識別子は英語を使用する。
